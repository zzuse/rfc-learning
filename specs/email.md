电子邮件（Email）相关的核心 RFC 标准主要涉及传输、消息格式、MIME 扩展和安全性。最关键的包括：负责传输的 RFC 5321（SMTP）和负责内容格式的 RFC 5322（邮件格式），它们取代了旧的 RFC 821/822。 
核心邮件 RFC 标准一览：
传输协议 (SMTP):
RFC 5321 (SMTP): 定义了简单邮件传输协议，取代了 RFC 2821 和 RFC 821。
RFC 1123: 对因特网主机进行规定，更新了 SMTP。
邮件消息格式:
RFC 5322 (Internet Message Format): 定义了电子邮件的文本消息格式，取代了 RFC 2822 和 RFC 822。
RFC 6854: 更新了 RFC 5322，允许在“发件人”头字段中使用分组语法。
MIME (多用途邮件扩展):
RFC 2045 - 2047: 定义了邮件正文格式、媒体类型以及非 ASCII 字符的头部扩展。
RFC 2231: 用于 MIME 参数值和编码字的扩展。
其他相关 RFC:
RFC 2142: 定义了通用的服务和角色邮箱名称（如 postmaster, abuse）。
RFC 6530: 国际化电子邮件概述（支持非英文邮件地址）。
RFC 1035: 域名系统 (DNS) 规范，用于 MX 记录解析
